通过MRC理解iOS内存管理本质
Publish: 2017/7/16   
  • iOS

在 Xcode 4.2 之前还没引入 ARC 的时候，使用的是 MRC 来管理iOS项目的内存。

MRC(MannulReference Counting)
ARC(Automatic Reference Counting)

ARC和MRC原理一样，只是ARC下无需手工retain或者release，系统会自动管理已创建的对象或内存空间。使用ARC可以省去许多繁复的操作，而使用MRC可以按自己期望精确地管理对象及内存的使用，同时通过使用MRC也易于理解iOS项目内存管理的本质。

在TARGETS的Build Settings中将LLVM编辑器设置ARC为无效就可以使用MRC管理内存了。






内存管理基本规则

💡 自己生成的对象，自己所持有。
(You own any object you create.)
💡 非自己生成的对象，自己也能持有。
(You can take ownership of an object using retain.)
💡 不再需要持有对象时释放。
(When you no longer need it, you must relinquish ownership of an object you own.)
💡 非自己持有的对象无法释放。
(You must not relinquish ownership of an object you do not own.)

相关方法


















📎      自己生成的对象，自己所持有。

当一个对象由alloc/new/copy/mutableCopy其中一种方法生成，表示自己生成并持有对象，系统会给这个对象分配一块内存，此时这个对象的引用计数由0变为1，表示有一个对象或指针持有这块内存。

上代码：

UIView *view1 = [[UIView alloc] init];
NSLog(@"%lu",[view1 retainCount]);

对象的引用计数可以用retainCount实例方法取得(此处用retainCount方法举例，方便理解，实际上这个方法不被推荐使用)，可以看到此时打印的view1的retainCount=1。

📎      非自己生成的对象，自己也能持有。

当有另一个对象或指针指向、持有这块内存，原对象的引用计数在原来的基础上+1。

UIView *view1 = [[UIView alloc] init];
UIView *view2 = view1;
[view2 retain];
NSLog(@"%lu",[view1 retainCount]);

打印的view1的retainCount=2。
retain方法用来持有非自己生成并持有的对象，如果没有[view2 retain],则view2并未拥有view1的所有权，此时打印的view1的retainCount仍然为1。

📎      不再需要持有对象时释放。

当一个对象不再需要使用时，必须将该对象释放,释放完成后与该对象相关的引用计数-1。

UIView *view1 = [[UIView alloc] init];
[view1 release];

对象一经释放不可再访问，当一个对象的引用计数减到0时，系统会调用dealloc方法废弃该对象，其占用的内存会被系统回收。

需要注意的是：

UIView *view1 = [[UIView alloc] init];
UIView *view2 = view1;
[view1 release];
[view2 setBackgroundColor:[UIColor blackColor]];

上述情况程序会崩溃，原因是view1被release，经过dealloc后内存被系统回收，view1不再持有内存。而view2仍然指向这块内存的地址，其所指的位置已经被销毁，所以此时的view2是野指针，它可以指向一块地方，但是对它进行任何操作，程序就会崩溃。
此时如果view2还需要,可以给它赋一个持有内存的新值以对view2进行操作，或者暂时先给它一个nil。

UIView *view3 = [[UIView alloc] init];
view2 = view3;
[view2 setFrame:CGRectMake(50, 50, 100, 100)];

//或先给它一个nil，之后再进行以上类似操作。
view2 = nil;

或者一开始就retain：

UIView *view1 = [[UIView alloc] init];
UIView *view2 = view1;
[view2 retain];
[view1 release];
[view2 setBackgroundColor:[UIColor blackColor]];

如此就算view1释放，view2还是持有这块内存，这样就可以继续对view2进行操作啦！操作一波之后不再需要view2了记得release掉它。

📎      非自己持有的对象无法释放。

非alloc/new/copy/mutableCopy方法生成并持有的，或使用retain方法持有的对象，不能进行释放。

UIView *view1 = [[UIView alloc] init];
[view1 release];//你已经失去我了ㄟ( ▔, ▔ )ㄏ
[view1 release];//再次释放已经废弃的对象或者说非自己持有的对象，程序就会崩溃。

NSString *str = [NSMutableString stringWithFormat:@"Hello World!"];
[str release];//释放了非自己持有的对象，程序崩溃。

一般用alloc/new/copy/mutableCopy方法以外的方法生成的对象都被声明了autorelease，autorelease会自动延迟释放内存，不用手动释放，系统会在合适的时候去释放掉它。
使用alloc/new/copy/mutableCopy方法创建的及retain方法持有的对象才需要去考虑引用计数。

最后

谁 alloc/new/copy/mutableCopy，谁 release。
谁 retain，谁 release。
即：
A alloc/new/copy/mutableCopy的次数必须等于 A release/autorelease的次数。
B retain的次数必须等于 B release的次数。
