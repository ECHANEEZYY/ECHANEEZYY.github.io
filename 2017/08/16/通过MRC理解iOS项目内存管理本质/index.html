通过MRC理解iOS项目内存管理本质

2017-08-16 | 学习 |

在Xcode4.2之前还没引入ARC(Automatic Reference Counting)的时候，使用的是MRC(MannulReference Counting)来管理iOS项目的内存。
ARC和MRC原理一样，只是ARC下无需手工retain或者release，系统会自动管理已创建的对象或内存空间。使用ARC的同时，MRC也要了解。

在TARGETS的Build Settings中将LLVM编辑器设置为ARC无效就可以使用MRC管理内存了。



内存管理基本规则

  • 自己生成的对象，自己所持有。(You own any object you create.)
  • 非自己生成的对象，自己也能持有。(You can take ownership of an object using retain.)
  • 不再需要持有对象时释放。(When you no longer need it, you must relinquish ownership of an object you own.)

  • 非自己持有的对象无法释放。(You must not relinquish ownership of an object you do not own.)

📎 自己生成的对象，自己所持有。

📎 非自己生成的对象，自己也能持有。

当有另一个对象或指针指向、持有这块内存，原对象的引用计数在原来的基础上+1。

打印的view1的retainCount=2。
retain方法用来持有非自己生成并持有的对象，如果没有[view2 retain],则view2并未拥有view1的所有权，此时打印的view1的retainCount仍然为1。

📎 不再需要持有对象时释放。

当一个对象不再需要使用时，必须将该对象释放,释放完成后与该对象相关的引用计数-1。

对象一经释放不可再访问，当一个对象的引用计数减到0时，系统会调用dealloc方法废弃该对象，其占用的内存会被系统回收。

📎 非自己持有的对象无法释放。

非alloc/new/copy/mutableCopy方法生成并持有的，或使用retain方法持有的对象，不能进行释放。

一般用alloc/new/copy/mutableCopy方法以外的方法生成的对象都被声明了autorelease，autorelease会自动延迟释放内存，不用手动释放，系统会在合适的时候去释放掉它。
使用alloc/new/copy/mutableCopy方法创建的及retain方法持有的对象才需要去考虑引用计数。

最后

谁 alloc/new/copy/mutableCopy，谁 release。
谁 retain，谁 release。
即：
A alloc/new/copy/mutableCopy的次数必须等于 A release/autorelease的次数。
B retain的次数必须等于 B release的次数。
